# Spec: S0.2 — CI Pipeline

## Goal

Set up GitHub Actions CI so that every PR has automated feedback before merge.

## What to Build

### `.github/workflows/ci.yml`

Single workflow, triggered on:
- Push to any branch
- Pull request to `main`

**Jobs:**

1. **lint**
   - `cargo fmt --check`
   - `cargo clippy --workspace -- -D warnings`
   - `forge fmt --check`

2. **test**
   - `cargo test --workspace`
   - `forge test`

3. **integration** (stub)
   - `echo "Integration tests — will be enabled when contracts are deployed"`
   - Kept as placeholder to be activated in S1a.1+

4. **e2e** (stub)
   - `echo "E2E tests — will be enabled when full flow exists"`
   - Kept as placeholder to be activated after Slice 1

### Branch protection (manual step)

Document in README or CLAUDE.md:
- Require CI pass before merge
- Require 1 approval
- No direct push to `main`

_(Branch protection is configured via GitHub UI, not automated)_

### justfile update

Ensure `just lint` and `just test` include both Cargo and Foundry commands.

## What NOT to Build

- Security review action (no code to scan yet)
- Claude Code Review action (no meaningful PRs yet)
- Release/deploy pipeline (nothing to release)
- Commitlint tooling (CLAUDE.md rules are sufficient)

## Validation

- Push a branch, open a PR → CI runs and passes
- `just ci` mirrors what GitHub Actions runs

## Handoff Prompt

```
Set up CI/CD for HardTrust.

Read the spec at docs/specs/s0.2-ci-pipeline.spec.md first.

Create a GitHub Actions workflow with lint and test jobs.
Add integration and e2e as stubs. Update justfile to include
Foundry commands. Run `just ci` to validate.
```
