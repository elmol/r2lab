# Spec: S0.7 — Solidity Static Analysis & Pre-Commit Validation

## Goal

Add Solhint and Aderyn to the HardTrust project so Solidity code is checked for style, best practices, and security issues. Add a pre-commit validation rule to CLAUDE.md so Claude Code always runs checks and fixes issues before committing.

## What to Build

### `contracts/.solhint.json`

```json
{
  "extends": "solhint:recommended",
  "rules": {
    "gas-custom-errors": "error",
    "func-visibility": ["error", { "ignoreConstructors": true }],
    "compiler-version": ["error", ">=0.8.19"]
  }
}
```

Key rule: `gas-custom-errors` enforces custom errors over `require(condition, "string")` — cheaper gas and modern Solidity practice.

### `justfile` updates

Add to existing recipes:

```just
lint:
    @cargo fmt --check 2>/dev/null || echo "No workspace members to lint"
    @cargo clippy --workspace -- -D warnings 2>/dev/null || echo "No workspace members to check"
    cd contracts && forge fmt --check
    cd contracts && npx solhint 'src/**/*.sol'
    cd contracts && aderyn .
```

Note: `aderyn` outputs a report to stdout. If it finds high/critical issues, the implementer should fix them. Aderyn does not exit non-zero on findings by default — the report is informational for now.

### `.github/workflows/ci.yml` updates

Add Solhint to the **lint** job:

```yaml
- name: Solhint
  run: cd contracts && npx solhint 'src/**/*.sol'
```

Aderyn runs locally only for now (installation in CI is heavier — defer to S1c).

### CLAUDE.md — Pre-Commit Validation section

Add this block to CLAUDE.md:

```markdown
## Pre-Commit Validation

Before EVERY `git commit`, run `just ci`. If it fails:

1. Run `cargo fmt` and `forge fmt` to auto-fix formatting
2. Fix any clippy or solhint warnings in the code
3. Run `just ci` again to confirm all checks pass
4. Only then proceed with the commit

Never commit with known lint or test failures. If `just ci` fails 3 times after attempting fixes, stop and report the issue to the human.
```

This replaces any existing "run `just ci` before pushing" instruction. The key change: validation happens before **every commit**, not just before PRs.

### Fix existing contract code

After adding the tools, run them against all contracts in `contracts/src/` and fix all findings.

### Output files

- `contracts/.solhint.json`
- `contracts/package.json` (for solhint dependency)
- Updated `justfile`
- Updated `.github/workflows/ci.yml`
- Updated `CLAUDE.md` (pre-commit validation section)
- Any Solidity files in `contracts/src/` and `contracts/test/` that need fixes

## What NOT to Build

- No Aderyn in CI (defer to S1c — requires Rust toolchain in CI runner)
- No Slither (Python dependency, too heavy for walking skeleton)
- No git pre-commit hooks (CLAUDE.md instruction is sufficient for Claude Code; hooks are for human devs, defer to S1c if needed)
- No custom Solhint plugins

## Validation

```bash
cd contracts && npx solhint 'src/**/*.sol'
# Should pass with no errors

cd contracts && aderyn .
# Should report no high/critical issues

just ci
# Should pass (lint + test)
```
