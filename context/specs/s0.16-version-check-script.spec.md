# S0.16: Extract Version Check to Script

**Type:** Refactor — simplify S0.15 version guard
**Depends on:** S0.15 merged

---

## Problem

The version guard in `release.yml` uses inline Python (`python3 -c "import sys,json..."`). This is:
- Harder to read and maintain in YAML
- Not runnable locally to debug
- Using Python for something bash handles fine

---

## What to Build

### 1. Create `scripts/check-version.sh`

Reads the workspace version from root `Cargo.toml` (one grep, no Python) and compares it to the tag.

```bash
#!/usr/bin/env bash
# check-version.sh — Verify Cargo.toml version matches the release tag.
#
# Usage: RELEASE_VERSION=v0.2.0 bash scripts/check-version.sh
#
# Called by release.yml build jobs. Can also be run locally to debug.

set -euo pipefail

if [ -z "${RELEASE_VERSION:-}" ]; then
  echo "ERROR: RELEASE_VERSION is not set." >&2
  echo "       Usage: RELEASE_VERSION=v0.2.0 bash scripts/check-version.sh" >&2
  exit 1
fi

# Read version from workspace Cargo.toml — works because version.workspace = true
CARGO_VERSION=$(grep '^version' Cargo.toml | head -1 | sed 's/version = "\(.*\)"/\1/')
TAG_VERSION="${RELEASE_VERSION#v}"   # strip leading 'v'

if [ "${CARGO_VERSION}" != "${TAG_VERSION}" ]; then
  echo "ERROR: Cargo.toml version (${CARGO_VERSION}) does not match tag (${TAG_VERSION})" >&2
  echo "       Run 'cargo release' to cut releases — do not push tags manually." >&2
  exit 1
fi

echo "Version check OK: ${CARGO_VERSION}"
```

Make it executable: `git update-index --chmod=+x scripts/check-version.sh`

### 2. Update `.github/workflows/release.yml`

Replace the inline Python version guard step in each build job with:

```yaml
- name: Verify Cargo version matches tag
  run: bash scripts/check-version.sh
```

That's it. One line.

### 3. Add justfile recipe for local use

```justfile
# Verify Cargo.toml version matches RELEASE_VERSION (local debug)
check-version:
    RELEASE_VERSION={{release_version}} bash scripts/check-version.sh
```
