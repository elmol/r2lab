# Spec: S0.1 — Repository Initialization

## Goal

Create the HardTrust repository with the minimum structure needed to start implementing Slice 1.

## What to Create

1. **Git repository** initialized with a `main` branch
2. **README.md** — one-liner: "HardTrust — DePIN device identity and attestation system"
3. **CLAUDE.md** — project rules for Claude Code:
   - Read specs from `docs/specs/` before implementing
   - Create an ADR when making a significant technical decision
   - Conventional commits
   - Run `just ci` before pushing
   - No direct pushes to main (PRs only)
4. **Cargo workspace** — empty members list for now (members added as crates are created)
5. **justfile** — minimal:
   - `just test` → `cargo test --workspace`
   - `just lint` → `cargo fmt --check && cargo clippy --workspace -- -D warnings`
   - `just ci` → `just lint && just test`
6. **contracts/** — empty Foundry project (`forge init contracts --no-git`)
7. **docs/adr/** — directory ready for ADRs
8. **docs/specs/** — directory where specs from r2lab will be copied
9. **.gitignore** — Rust + Foundry ignores

## What NOT to Create

- No `device/`, `attester/`, `common/`, `webapp/` crates yet
- No CI/CD pipeline yet (next spec)
- No ADRs yet (next spec)
- No tests (nothing to test yet)

## Validation

- `cargo build` succeeds (empty workspace is valid)
- `forge build` succeeds in `contracts/`
- `just ci` runs without errors
- Repository has a clean first commit

## Handoff Prompt

```
Create a new Rust + Foundry monorepo for HardTrust, a DePIN device identity
and attestation system.

Read the spec at docs/specs/s0.1-repo-initialization.spec.md for exact
requirements. Create only what the spec says — nothing more.

After setup, run `just ci` to validate.
```
