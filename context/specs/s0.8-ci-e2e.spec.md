# Spec: S0.8 — CI End-to-End Tests

## Goal

Ensure the full end-to-end flow (register → emit → verify) runs automatically on every push and pull request. If The Wire breaks, the team knows immediately — no manual checking required.

## Context

The CI pipeline (S0.2) currently runs lint, unit tests, and Solidity tests on every push. The e2e job exists but is a placeholder stub that prints a message and passes without testing anything. The e2e-the-wire script (`scripts/e2e-the-wire.sh`) already works locally and covers both VERIFIED and UNVERIFIED cases.

## Acceptance Criteria

- [ ] When I push code to any branch, the CI pipeline runs the full end-to-end flow automatically — I don't have to remember to run it locally
- [ ] When the e2e flow succeeds, the CI job passes — I see a green check on my commit
- [ ] When any part of the e2e flow breaks (register, emit, verify, or the UNVERIFIED case), the CI job fails — I see a red X and know something is wrong before merging
- [ ] The e2e CI job uses the same script I run locally (`scripts/e2e-the-wire.sh`) — there is no separate "CI version" that could drift
- [ ] The CI handles its own setup (Anvil, Foundry, Rust) — I don't need to configure anything special for e2e to run
- [ ] If Anvil fails to start or hangs, the job times out and fails instead of running forever

## What NOT to Build

- No new e2e scripts — use the existing `scripts/e2e-the-wire.sh`
- No separate CI-only test infrastructure
- No deployment to external networks (Anvil only)
- No changes to the e2e script itself (unless the CI environment requires minor adjustments)

## Validation

After implementation:
- Push a commit → CI runs → e2e job passes (green)
- Intentionally break something (e.g., change the contract) → push → e2e job fails (red)

## Handoff Prompt

```
Replace the CI e2e stub with a real job that runs scripts/e2e-the-wire.sh.

Read the spec at docs/specs/s0.8-ci-e2e.spec.md first.

The e2e job in .github/workflows/ci.yml is currently a placeholder.
Replace it with a real job that:
- Sets up Rust, Foundry (includes Anvil), and just
- Runs: just e2e-the-wire
- Fails if the script exits non-zero
- Has a reasonable timeout (10 minutes max)

The script already handles Anvil startup/cleanup internally.
Do not modify scripts/e2e-the-wire.sh unless the CI environment requires it.

Respect the "What NOT to Build" section strictly.
Branch: feat/s0.8-ci-e2e
```
