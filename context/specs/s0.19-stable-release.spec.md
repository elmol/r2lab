# S0.19: Stable Release v0.1.0

**Type:** Release
**Depends on:** S0.14 (CLI version), S0.15 (cargo-release), S0.18 (install trap fix)

---

## Problem

The current release is `v0.1.0-rc8` (pre-release). It was built **before** S0.14 (CLI `--version` flag) and S0.18 (install script trap fix) were merged. Users installing via `install-device.sh` or `install-attester.sh` get binaries without `--version` support, and the install scripts show an error on exit.

All fixes are now merged to `main`. A stable `v0.1.0` release will include everything.

## What to Do

### Step 1 — Verify main is clean

```bash
just ci
device --version   # should print "device 0.1.0"
attester --version # should print "attester 0.1.0"
```

### Step 2 — Cut the release

The workspace version is already `0.1.0` in `Cargo.toml`. Use `cargo release` to tag and push:

```bash
# Dry run first
cargo release 0.1.0 --no-confirm --dry-run

# If dry run looks good, cut the release
cargo release 0.1.0 --no-confirm --execute
```

This will:
1. Run `cargo test --workspace` (pre-release hook in `release.toml`)
2. Create git tag `v0.1.0`
3. Push commit + tag to origin
4. GitHub Actions `release.yml` triggers: builds 3 binaries (device ARMv7, attester Linux, attester macOS) and creates the GitHub Release

### Step 3 — Verify the release

After CI completes (~5 min):

```bash
# Check GitHub Release exists
gh release view v0.1.0

# Test install scripts
curl -fsSL https://raw.githubusercontent.com/elmol/hardtrust/main/install-attester.sh | bash
attester --version  # should print "attester 0.1.0"
```
