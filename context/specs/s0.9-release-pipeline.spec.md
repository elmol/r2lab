# S0.9: Release Pipeline — GitHub Actions Cross-Compilation

**Type:** Infrastructure
**Story Reference:** N/A — delivery infrastructure
**Depends on:** All S1 + S1-Debt specs Implemented

---

## Goal

Produce downloadable binaries for all target platforms automatically when a git tag is pushed. The developer's workflow is:

```bash
git tag v0.1.0
git push origin v0.1.0
# GitHub Actions builds 3 binaries, creates GitHub Release, uploads artifacts
```

No local cross-compilation tooling is required on developer machines.

---

## Target Matrix

| Binary | Target triple | Platform | Runner | Method |
|--------|--------------|----------|--------|--------|
| `device` | `armv7-unknown-linux-musleabihf` | Raspbian Bullseye ARMv7 | `ubuntu-latest` | `cargo-zigbuild` |
| `attester` | `x86_64-unknown-linux-gnu` | Ubuntu x86_64 | `ubuntu-latest` | native build |
| `attester` | `aarch64-apple-darwin` | macOS Apple Silicon | `macos-latest` | native build |

**Why musl for device:** Raspbian Bullseye ships glibc 2.31. A glibc-linked binary built on a newer CI runner would fail with `GLIBC not found`. Musl produces a fully static binary with zero runtime dependencies.

**Why gnu for attester Linux:** Server Ubuntu has current glibc. Alloy's RPC uses `getaddrinfo` — glibc's DNS resolver has broader compatibility than musl's stub resolver.

**Why cargo-zigbuild:** Uses Zig as a cross C linker. No Docker dependency. Installs in seconds via `taiki-e/install-action`. Handles musl targets reliably from any Linux host.

## What to Build

### 1. Create `.github/workflows/release.yml`

Full content:

```yaml
name: Release

on:
  push:
    tags:
      - "v*.*.*"

env:
  CARGO_TERM_COLOR: always
  RELEASE_VERSION: ${{ github.ref_name }}

jobs:
  # -------------------------------------------------------
  # Job 1: device binary — ARMv7 musl static for Raspbian 11
  # -------------------------------------------------------
  build-device-armv7:
    name: "device / armv7-unknown-linux-musleabihf"
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - uses: foundry-rs/foundry-toolchain@v1

      - name: Build contracts (ABI required by protocol crate)
        run: cd contracts && forge build

      - uses: dtolnay/rust-toolchain@stable
        with:
          targets: armv7-unknown-linux-musleabihf

      - name: Install cargo-zigbuild
        uses: taiki-e/install-action@v2
        with:
          tool: cargo-zigbuild

      - name: Install Zig
        uses: mlugg/setup-zig@v1

      - name: Build device (ARMv7 musl)
        run: |
          cargo zigbuild \
            --release \
            --package device \
            --target armv7-unknown-linux-musleabihf

      - name: Rename and checksum
        run: |
          ARTIFACT="device-${RELEASE_VERSION}-armv7-unknown-linux-musleabihf"
          cp target/armv7-unknown-linux-musleabihf/release/device "${ARTIFACT}"
          sha256sum "${ARTIFACT}" > "${ARTIFACT}.sha256"

      - uses: actions/upload-artifact@v4
        with:
          name: device-armv7
          path: |
            device-${{ env.RELEASE_VERSION }}-armv7-unknown-linux-musleabihf
            device-${{ env.RELEASE_VERSION }}-armv7-unknown-linux-musleabihf.sha256
          if-no-files-found: error

  # -------------------------------------------------------
  # Job 2: attester — x86_64 Linux (Ubuntu), native build
  # -------------------------------------------------------
  build-attester-linux:
    name: "attester / x86_64-unknown-linux-gnu"
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - uses: foundry-rs/foundry-toolchain@v1

      - name: Build contracts
        run: cd contracts && forge build

      - uses: dtolnay/rust-toolchain@stable
        with:
          targets: x86_64-unknown-linux-gnu

      - name: Build attester (x86_64 Linux)
        run: |
          cargo build \
            --release \
            --package attester \
            --target x86_64-unknown-linux-gnu

      - name: Rename and checksum
        run: |
          ARTIFACT="attester-${RELEASE_VERSION}-x86_64-unknown-linux-gnu"
          cp target/x86_64-unknown-linux-gnu/release/attester "${ARTIFACT}"
          sha256sum "${ARTIFACT}" > "${ARTIFACT}.sha256"

      - uses: actions/upload-artifact@v4
        with:
          name: attester-linux
          path: |
            attester-${{ env.RELEASE_VERSION }}-x86_64-unknown-linux-gnu
            attester-${{ env.RELEASE_VERSION }}-x86_64-unknown-linux-gnu.sha256
          if-no-files-found: error

  # -------------------------------------------------------
  # Job 3: attester — macOS ARM64 (Apple Silicon), native
  # -------------------------------------------------------
  build-attester-macos:
    name: "attester / aarch64-apple-darwin"
    runs-on: macos-latest
    steps:
      - uses: actions/checkout@v4

      - uses: foundry-rs/foundry-toolchain@v1

      - name: Build contracts
        run: cd contracts && forge build

      - uses: dtolnay/rust-toolchain@stable
        with:
          targets: aarch64-apple-darwin

      - name: Build attester (macOS ARM64)
        run: |
          cargo build \
            --release \
            --package attester \
            --target aarch64-apple-darwin

      - name: Rename and checksum
        run: |
          ARTIFACT="attester-${RELEASE_VERSION}-aarch64-apple-darwin"
          cp target/aarch64-apple-darwin/release/attester "${ARTIFACT}"
          shasum -a 256 "${ARTIFACT}" > "${ARTIFACT}.sha256"

      - uses: actions/upload-artifact@v4
        with:
          name: attester-macos
          path: |
            attester-${{ env.RELEASE_VERSION }}-aarch64-apple-darwin
            attester-${{ env.RELEASE_VERSION }}-aarch64-apple-darwin.sha256
          if-no-files-found: error

  # -------------------------------------------------------
  # Job 4: Aggregate + publish GitHub Release
  # -------------------------------------------------------
  release:
    name: "Publish GitHub Release"
    runs-on: ubuntu-latest
    needs: [build-device-armv7, build-attester-linux, build-attester-macos]
    permissions:
      contents: write

    steps:
      - uses: actions/checkout@v4
        with:
          fetch-depth: 0   # full history needed for git log release notes

      - uses: actions/download-artifact@v4
        with:
          path: artifacts
          merge-multiple: true

      - name: Aggregate SHA256SUMS
        run: |
          cd artifacts
          cat *.sha256 | sort > SHA256SUMS
          cat SHA256SUMS

      - name: Generate release notes
        id: notes
        run: |
          PREVIOUS_TAG=$(git tag --sort=-version:refname | grep -v "^${RELEASE_VERSION}$" | head -1)
          if [ -z "${PREVIOUS_TAG}" ]; then
            RANGE="HEAD"
            HEADER="Initial release"
          else
            RANGE="${PREVIOUS_TAG}..HEAD"
            HEADER="Changes since ${PREVIOUS_TAG}"
          fi
          COMMITS=$(git log ${RANGE} --pretty=format:"- %s (%h)" --no-merges)
          cat > release_notes.md << EOF
          ## ${HEADER}

          ${COMMITS}

          ## Artifacts

          | File | Platform |
          |------|----------|
          | \`device-${RELEASE_VERSION}-armv7-unknown-linux-musleabihf\` | Raspberry Pi / Raspbian 11 ARMv7 |
          | \`attester-${RELEASE_VERSION}-x86_64-unknown-linux-gnu\` | Ubuntu / Linux x86_64 |
          | \`attester-${RELEASE_VERSION}-aarch64-apple-darwin\` | macOS Apple Silicon |

          Verify integrity with \`SHA256SUMS\` or individual \`.sha256\` sidecar files.

          ## Install

          \`\`\`bash
          curl -fsSL https://raw.githubusercontent.com/${{ github.repository }}/${RELEASE_VERSION}/install.sh | bash
          \`\`\`
          EOF

      - name: Create GitHub Release
        uses: softprops/action-gh-release@v2
        with:
          tag_name: ${{ github.ref_name }}
          name: "HardTrust ${{ github.ref_name }}"
          body_path: release_notes.md
          draft: false
          prerelease: ${{ contains(github.ref_name, '-') }}
          files: |
            artifacts/device-${{ env.RELEASE_VERSION }}-armv7-unknown-linux-musleabihf
            artifacts/device-${{ env.RELEASE_VERSION }}-armv7-unknown-linux-musleabihf.sha256
            artifacts/attester-${{ env.RELEASE_VERSION }}-x86_64-unknown-linux-gnu
            artifacts/attester-${{ env.RELEASE_VERSION }}-x86_64-unknown-linux-gnu.sha256
            artifacts/attester-${{ env.RELEASE_VERSION }}-aarch64-apple-darwin
            artifacts/attester-${{ env.RELEASE_VERSION }}-aarch64-apple-darwin.sha256
            artifacts/SHA256SUMS
```

### 2. Add `[profile.release]` to workspace `Cargo.toml`

Check if `[profile.release]` already exists. If not, append:

```toml
[profile.release]
strip = true        # remove debug symbols — smaller binaries
lto = "thin"        # thin LTO: good compile time / size tradeoff
codegen-units = 1   # enables full cross-crate inlining
opt-level = 3
panic = "abort"     # removes unwinder — safe for CLI, important for musl static
```

`panic = "abort"` is correct for CLI binaries. It removes the unwinding machinery which is especially beneficial for musl static binaries.

### 3. Add release recipes to `justfile`

Append to the existing `justfile`:

```justfile
# ---------------------------------------------------------------------------
# Release recipes
# ---------------------------------------------------------------------------
release_version := "v0.1.0"

# Build attester for the host platform (release mode)
build-release:
    cargo build --release --package attester
    @echo "Binary: target/release/attester"

# Stage release artifacts for current host into ./dist/ (for local testing)
dist: forge-build
    mkdir -p dist
    cargo build --release --package attester --target x86_64-unknown-linux-gnu 2>/dev/null || \
        cargo build --release --package attester
    ARTIFACT="attester-{{release_version}}-$(rustc -vV | grep host | awk '{print $2}')" && \
        cp target/release/attester "dist/$${ARTIFACT}" && \
        (sha256sum "dist/$${ARTIFACT}" > "dist/$${ARTIFACT}.sha256" 2>/dev/null || \
         shasum -a 256 "dist/$${ARTIFACT}" > "dist/$${ARTIFACT}.sha256")
    @echo "Artifacts in ./dist/"
    @ls -lh dist/
```

## What NOT to Build

- Do not modify the existing `ci.yml` workflow
- Do not add macOS Intel (x86_64-apple-darwin) target — not in scope for v0.1.0
- Do not add Linux ARM64 (aarch64-unknown-linux-gnu) — not in scope for v0.1.0
- Do not set up Homebrew tap
- Do not add `cargo-dist` tooling
