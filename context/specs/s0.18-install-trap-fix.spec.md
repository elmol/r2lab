# S0.18: Fix Install Scripts — Trap Unbound Variable Error

**Type:** Bug fix
**Depends on:** S0.13

---

## Problem

Both `install-device.sh` and `install-attester.sh` fail with:

```
bash: línea 1: tmp: variable sin asignar
```

The error occurs on exit. The scripts use `set -euo pipefail` (`-u` = error on unbound variables). In `main()`, `tmp` is declared as `local` and a trap is set:

```bash
local tmp; tmp="$(mktemp -d)"; trap 'rm -rf "${tmp}"' EXIT
```

When the trap fires at script exit, `tmp` is out of scope (it was local to `main()`), so `${tmp}` is unbound and `-u` causes the error.

The installation itself succeeds — this is a cosmetic error on exit — but it confuses users and signals something went wrong.

## Fix

Move `tmp` to module scope (before `main`) so it survives after `main()` returns. Apply to both scripts identically.

### install-device.sh — line 71

**Before:**
```bash
main() {
  ...
  local tmp; tmp="$(mktemp -d)"; trap 'rm -rf "${tmp}"' EXIT
  ...
}
```

**After:**
```bash
TMP_DIR=""
cleanup() { [ -n "${TMP_DIR}" ] && rm -rf "${TMP_DIR}"; }
trap cleanup EXIT

main() {
  ...
  TMP_DIR="$(mktemp -d)"
  ...
  # Use ${TMP_DIR} instead of ${tmp} everywhere in main()
}
```

### install-attester.sh — same pattern

Apply the identical fix: replace `local tmp` + inline trap with module-scope `TMP_DIR` + `cleanup()` function.

### Changes in both scripts

| Line | Before | After |
|------|--------|-------|
| Module scope | — | `TMP_DIR=""` + `cleanup()` + `trap cleanup EXIT` |
| Inside `main()` | `local tmp; tmp="$(mktemp -d)"; trap 'rm -rf "${tmp}"' EXIT` | `TMP_DIR="$(mktemp -d)"` |
| All references | `${tmp}` | `${TMP_DIR}` |
