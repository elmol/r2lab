# S0.13: Fix Install Scripts — Pre-release Version Resolution

**Type:** Bug fix
**Depends on:** S0.12

---

## Problem

`/releases/latest` GitHub API returns 404 when no stable release exists — it ignores pre-releases (tags containing `-` like `v0.1.0-rc8`). Both `install-device.sh` and `install-attester.sh` fail with `ERROR: Could not resolve latest release.`

## Fix

Replace the `resolve_version()` function in both scripts to use `/releases` (all releases) and take the first one when `HARDTRUST_VERSION` is not set:

```bash
resolve_version() {
  [ -n "${HARDTRUST_VERSION:-}" ] && { echo "${HARDTRUST_VERSION}"; return; }

  local v
  # Use /releases (not /releases/latest) so pre-releases are included
  v=$(curl -fsSL "https://api.github.com/repos/${REPO}/releases" \
    | grep '"tag_name"' \
    | head -1 \
    | sed 's/.*"tag_name": *"\([^"]*\)".*/\1/')

  [ -n "${v}" ] || {
    echo "ERROR: Could not resolve latest release." >&2
    echo "       Set HARDTRUST_VERSION=v0.1.0-rc8 or check https://github.com/${REPO}/releases" >&2
    exit 1
  }
  echo "${v}"
}
```

Apply this change to both `install-device.sh` and `install-attester.sh`.

## Validation Criteria

- [ ] `curl -fsSL .../install-attester.sh | bash` works when only pre-releases exist
- [ ] `HARDTRUST_VERSION=v0.1.0-rc8 curl -fsSL .../install-attester.sh | bash` still works
- [ ] After a stable `v0.1.0` tag exists, installs the stable version by default
