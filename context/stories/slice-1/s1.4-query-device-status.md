# S1.4 — Query Device Status via CLI

## User Story
As anyone, I query the registration status of a device via CLI so that I can check whether a device is attested in the network.

## Acceptance Criteria

**AC-1: Query a registered device**
- Given a device that has been registered on-chain
- When the user runs the query command with the device's serial or address
- Then the CLI prints: registered status, attester address, and attestation timestamp

**AC-2: Query an unregistered device**
- Given a serial or address that has NOT been registered
- When the user runs the query command
- Then the CLI prints: "Not registered" or equivalent clear message

## Edge Cases

| Scenario | Expected Behavior |
|----------|-------------------|
| Invalid serial or address format | Clear error before querying |
| RPC endpoint not reachable | Clear error indicating connection failure |

## Dependencies
- **Upstream:** S1.2 (contract deployed), S1.3 (device registered, for testing the registered case)
- **Downstream:** S1.6 (verify uses this query as part of verification)
- **Shared:** `common/` (ABI bindings), `attester/` crate

## Out of Scope
- Web portal verification (S2.3)
- Device detail view (S3.3)
- Human-readable device names (R2.2)

## Definition of Done
- `attester query --serial <serial>` returns correct status for both registered and unregistered devices
- Tests pass in CI against Anvil
- Linting and formatting pass
