# S2a.4-V2: Embedded Release Hashes — Default Environment Verification

**Type:** Enhancement
**Story:** [S2a.4 Capture Environment Attestation](../stories/slice-2/s2a.4-capture-environment-attestation.md)
**Depends on:** S2a.4-V1 (environment attestation)

---

## Problem

The `--release-hashes` flag in `attester verify` is optional. If nobody passes it, environment attestation data is printed but never verified. This makes the feature useless by default.

## What to Build

### Step 1 — Embed SHA256SUMS in the attester binary at compile time

In `attester/build.rs` or directly in the source:

```rust
// attester/src/lib.rs (or wherever verify logic lives)
const RELEASE_HASHES: &str = include_str!("../../SHA256SUMS");
```

The `SHA256SUMS` file already exists in the repo root (generated by the release pipeline). It contains lines like:

```
sha256:<hex>  device-armv7
sha256:<hex>  terrascope-capture.sh
sha256:<hex>  attester-linux-x86_64
```

### Step 2 — Change default verify behavior

The attester verify command changes from:

- **Before:** `--release-hashes` optional, no env check by default
- **After:** env check runs by default using embedded hashes. `--release-hashes <path>` overrides with a custom file. `--skip-env-check` disables it entirely.

```rust
Verify {
    #[arg(long)]
    file: PathBuf,
    /// Override embedded release hashes with a custom file
    #[arg(long)]
    release_hashes: Option<PathBuf>,
    /// Skip environment verification entirely
    #[arg(long, default_value_t = false)]
    skip_env_check: bool,
}
```

### Step 3 — Verify logic

```
fn verify_environment(capture: &Capture, hashes_content: &str) -> EnvResult {
    // Parse hashes_content (same format as before)
    // Compare capture.environment.script_hash against "capture.sh" entry
    // Compare capture.environment.binary_hash against "device" entry
    // Return MATCH/MISMATCH per field
}

// In verify handler:
if !skip_env_check {
    let hashes = match release_hashes {
        Some(path) => std::fs::read_to_string(path)?,
        None => RELEASE_HASHES.to_string(),  // embedded default
    };
    verify_environment(&capture, &hashes);
}
```

### Step 4 — Ensure SHA256SUMS exists for dev/test

For local development and tests, the SHA256SUMS file may not exist or have placeholder values. Add a fallback:

```rust
const RELEASE_HASHES: &str = if cfg!(debug_assertions) {
    "" // empty in debug builds — skip env check gracefully
} else {
    include_str!("../../SHA256SUMS")
};
```

If the embedded hashes are empty, print a warning and skip env check (same as `--skip-env-check`).

## Output format

Default verify (no flags needed):

```
Signature:   VERIFIED
Environment:
  script_hash:  sha256:aaa111... → MATCH (terrascope-capture.sh)
  binary_hash:  sha256:bbb222... → MATCH (device-armv7)
  hw_serial:    10000000abcdef01
  camera_info:  mmal service 16.1
```

With `--skip-env-check`:

```
Signature:   VERIFIED
```

| `SHA256SUMS` (repo root) | Ensure exists (may need placeholder for dev builds) |

## Tests (TDD — write first)

1. **Default verify includes env check** — verify without any flags checks environment against embedded hashes
2. **Override with --release-hashes** — custom file takes precedence over embedded
3. **--skip-env-check** — no environment output at all
4. **Debug build with empty hashes** — graceful skip with warning, no error
5. **Existing verify tests still pass** — no regressions

- [ ] `attester verify --file capture.json --release-hashes custom.txt` uses custom file
- [ ] `attester verify --file capture.json --skip-env-check` skips environment section
- [ ] Debug builds handle missing SHA256SUMS gracefully
