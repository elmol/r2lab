# Handoff: S0.15 — Versioning Workflow

## Context

Cargo.toml version is hardcoded and never updated on release. Binaries report wrong version. Fix with workspace version inheritance + cargo-release for one-command releases.

## Spec

Read `docs/specs/s0.15-versioning-workflow.spec.md` for full details.

## Task

### Step 1 — Workspace version in root `Cargo.toml`
Add to `[workspace]` section (read the file first to avoid duplicating existing fields):
```toml
[workspace.package]
version = "0.1.0"
edition = "2021"
```

### Step 2 — Inherit in each crate
In `device/Cargo.toml`, `attester/Cargo.toml`, `protocol/Cargo.toml`, replace:
```toml
version = "0.1.0"
edition = "2021"
```
with:
```toml
version.workspace = true
edition.workspace = true
```

### Step 3 — Create `release.toml` at repo root
```toml
publish = false
tag-name = "v{{version}}"
tag-message = "HardTrust {{version}}"
pre-release-commit-message = "chore: release {{version}}"
pre-release-hook = ["cargo", "test", "--workspace"]
push = true
```

### Step 4 — Add version guard to `.github/workflows/release.yml`
In each of the 3 build jobs, add this step after `actions/checkout@v4`:
```yaml
- name: Verify Cargo version matches tag
  run: |
    CARGO_VERSION=$(cargo metadata --no-deps --format-version 1 \
      | python3 -c "import sys,json; pkgs=json.load(sys.stdin)['packages']; \
        print(next(p['version'] for p in pkgs if p['name']=='device'))")
    TAG_VERSION="${RELEASE_VERSION#v}"
    if [ "${CARGO_VERSION}" != "${TAG_VERSION}" ]; then
      echo "ERROR: Cargo.toml version (${CARGO_VERSION}) does not match tag (${TAG_VERSION})"
      echo "Use 'cargo release' to cut releases, not 'git tag' directly."
      exit 1
    fi
    echo "Version check OK: ${CARGO_VERSION}"
```

### Step 5 — Validate
- `cargo metadata --no-deps --format-version 1 | grep '"version"'` → all 3 crates show same version
- `cargo release patch --dry-run` runs without error (install first: `cargo install cargo-release`)
- `just ci` passes
