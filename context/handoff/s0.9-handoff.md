# Handoff: S0.9 — Release Pipeline

## Context

Tag-triggered GitHub Actions pipeline that cross-compiles 3 binaries and publishes a GitHub Release automatically. No local tooling required on developer machines.

## Spec

Read `docs/specs/s0.9-release-pipeline.spec.md` — contains the complete YAML for `release.yml`.

## Task

### 1. Create `.github/workflows/release.yml`
Copy the complete YAML from the spec verbatim. It contains 4 jobs: `build-device-armv7`, `build-attester-linux`, `build-attester-macos`, `release`.

### 2. Append `[profile.release]` to workspace `Cargo.toml`
Check if it already exists first. If not, add:
```toml
[profile.release]
strip = true
lto = "thin"
codegen-units = 1
opt-level = 3
panic = "abort"
```

### 3. Append release recipes to `justfile`
Add `build-release` and `dist` recipes from the spec.

### 4. Validate
Push a test tag `v0.1.0-test` and verify:
- All 4 CI jobs complete
- GitHub Release created with 7 artifacts (3 binaries + 3 `.sha256` + `SHA256SUMS`)
- `file device-*-armv7-*` → `ELF 32-bit LSB executable, ARM, statically linked`
- Delete test tag after validation: `git tag -d v0.1.0-test && git push origin --delete v0.1.0-test`
