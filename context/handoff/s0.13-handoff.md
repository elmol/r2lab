# Handoff: S0.13 — Install Script Fix (Pre-Release Support)

## Context

`/releases/latest` on the GitHub API returns 404 when only pre-releases exist (tags containing a hyphen like `v0.1.0-rc8`). Both `install-device.sh` and `install-attester.sh` used this endpoint, causing installation to fail silently.

## Spec

Read `docs/specs/s0.13-install-script-fix.spec.md` for full details.

## Task

In both `install-device.sh` and `install-attester.sh`, replace the `/releases/latest` API call with `/releases` and use `jq` to find the first matching tag:

```bash
# Before:
LATEST=$(curl -fsSL "https://api.github.com/repos/elmol/hardtrust/releases/latest" \
  | jq -r '.tag_name')

# After:
LATEST=$(curl -fsSL "https://api.github.com/repos/elmol/hardtrust/releases" \
  | jq -r '[.[] | select(.tag_name | test("^v[0-9]"))][0].tag_name')
```

Also ensure `HARDTRUST_VERSION` env var override works correctly when piped through bash:

```bash
# Correct usage:
curl -fsSL https://... | HARDTRUST_VERSION=v0.1.0-rc8 bash
# NOT:
HARDTRUST_VERSION=v0.1.0-rc8 curl -fsSL https://... | bash
```

## Branch

`fix/s0.13-install-prerelease`

## Acceptance

- `curl .../install-attester.sh | bash` resolves the latest release including pre-releases
- `curl .../install-attester.sh | HARDTRUST_VERSION=v0.1.0-rc8 bash` installs specific version
- No 404 errors from GitHub API
