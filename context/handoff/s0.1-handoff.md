# Handoff: S0.1 — Repository Initialization

## Pre-requisites

- `cargo` and `rustc` installed
- `forge` (Foundry) installed
- `just` (task runner) installed
- GitHub CLI (`gh`) authenticated

## Instructions

1. Create a new empty directory for the project
2. Open Claude Code in that directory
3. Paste the prompt below

## Prompt

```
Create a new repository for HardTrust — a DePIN device identity and attestation system.
Rust + Solidity (Foundry) monorepo.

Create EXACTLY the following — nothing more:

1. Initialize git repo with `main` branch

2. README.md:
   # HardTrust
   DePIN device identity and attestation system.

3. CLAUDE.md with these rules:
   - Read specs from `docs/specs/` before implementing any story
   - Create an ADR in `docs/adr/` when making a significant technical decision
   - Use conventional commits: feat:, fix:, chore:, docs:, test:
   - Run `just ci` before pushing
   - No direct pushes to main — use PRs

4. Cargo.toml — workspace with empty members:
   [workspace]
   members = []
   resolver = "2"

5. justfile with:
   - test: cargo test --workspace
   - lint: cargo fmt --check && cargo clippy --workspace -- -D warnings
   - ci: just lint && just test

6. contracts/ — initialize with `forge init contracts --no-git`

7. .gitignore — Rust (target/) + Foundry (contracts/out/, contracts/cache/) ignores

8. Empty directories (add .gitkeep to each):
   - docs/adr/
   - docs/specs/

DO NOT create: device/, attester/, common/, webapp/, CI pipelines, tests, or ADRs.

Validate:
- `cargo build` succeeds
- `cd contracts && forge build` succeeds
- `just ci` runs without errors

Create a single commit: "chore: initialize HardTrust repository"

Then create the GitHub repo and push:
- gh repo create elmol/hardtrust --public --source=. --push
```

## After execution

1. Verify the repo exists at github.com/elmol/hardtrust
2. Copy specs S0.2, S0.3, S0.4 to `docs/specs/` in the HardTrust repo for next steps
3. Return to r2lab with any feedback or issues
